WS (Working Directory) : Donde trabajamos SA (Stage Area) : donde reflejamos los snapshopts de cada archivo para la proxoma version del repo INDEX (.git) : repositorio La base de datos de git se separan e 4 tipos de objetos:

  • blob : son los archivos finales
  • tree : representan los directorios
  • commit : la representan que trees y blobs hay en tu WD y que estado de cambio tiene cada uno
  • tag : representan "versiones"

Why do you need that? They are hidden with purpose. List of ignored files and folders can be configured in Settings | Editor | File Types -> Ignored files and folders.
